## 1.1 红外传感器原理

> [!NOTE]
> 红外模块是"近距离感知"传感器，和超声波（中远距离）互补使用。

### ① 从 "夜视仪" 说起：先懂底层逻辑

红外传感器的工作方式和超声波完全不同。超声波是"主动发射再等回波"，红外模块则是**被动接收反射信号**。

生活类比：你在黑暗的房间打开手电筒照向墙壁——如果墙是白色的，光线强烈反射回来，你能看清；如果墙是黑色的，光线被吸收，你什么都看不到。

红外模块就是利用这个原理：
- 模块上的**红外发射管**持续发射红外光
- 遇到不同颜色的物体，反射回来的光强度不同
- **白色/浅色**：反射强 -> 输出 **低电平（0）**
- **黑色/深色**：反射弱 -> 输出 **高电平（1）**

> [!WARNING]
> 注意：不同模块的阈值电路不同，输出逻辑可能相反。QX_A11 小车上的模块是：**白=0，黑=1**。接线前务必用万用表或 LED 验证。

### ② 红外模块是怎么工作的？—— 结构拆解

常见的红外模块（如 TCRT5000）内部有两个管脚：

| 管脚 | 功能 | 说明 |
| :--- | :--- | :--- |
| VCC | 电源 | 通常 3.3V-5V |
| GND | 地 | 共地 |
| DO | 数字输出 | 比较器输出，0 或 1 |
| AO | 模拟输出 | 反射强度电压值（本实验用不到） |

核心是一个**比较器芯片（LM393）**：
- 红外发射管发光 -> 接收管收到反射光 -> 产生电压 -> 比较器与电位器设定的阈值比较 -> 输出高/低

> [!TIP]
> **电位器的作用**：模块上那个蓝色小旋钮就是电位器，用来调节检测距离（阈值）。顺时针调，灵敏度降低（能检测更远）；逆时针调，灵敏度提高（只能检测很近）。

### ③ 小车上用了几个红外模块？

QX_A11 小车上装了 **4 个红外传感器**，分为两组：

| 传感器 | 引脚 | 用途 |
| :--- | :--- | :--- |
| 左循迹 (left_led1) | P3^5 | 检测左线 |
| 右循迹 (right_led1) | P3^4 | 检测右线 |
| 左避障 (left_led2) | P3^6 | 检测左侧障碍物 |
| 右避障 (right_led2) | P3^7 | 检测右侧障碍物 |

> [!NOTE]
> 命名中的 `led1` / `led2` 是历史遗留命名，实际功能不是 LED，而是红外传感器。`led1` 系列用于循迹，`led2` 系列用于避障。

## 1.2 代码实现

### ① 引脚定义（对照 [QX_A11.h](code/QX_A11.h)）

```c
// QX_A11.h
sbit left_led1  = P3^5;   // 左循迹传感器
sbit right_led1 = P3^4;   // 右循迹传感器
sbit left_led2  = P3^6;   // 左避障传感器
sbit right_led2 = P3^7;   // 右避障传感器
```

> [!TIP]
> 4 个传感器全部接在 P3 口，不占用 P1（电机 PWM）和 P2（超声波），互不冲突。

### ② 循迹逻辑：4 种状态

核心思路：把左右传感器的读数拼成一个 **2 位二进制数**，根据这个数决定动作：

```c
// 核心逻辑拆解
char data1, data2, data3;
data2 = left_led1;    // 左：0=白线, 1=黑线
data3 = right_led1;   // 右：0=白线, 1=黑线
data1 = data2 * 10 + data3;  // 拼成两位数：10/11/01/00
```

| data1 值 | 二进制 | 左传感器 | 右传感器 | 小车动作 | 含义 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **11** | 1 1 | 黑线 | 黑线 | 前进 | 在中间，正常走 |
| **10** | 1 0 | 白线 | 黑线 | 右转 | 偏左了，往右纠 |
| **01** | 0 1 | 黑线 | 白线 | 左转 | 偏右了，往左纠 |
| **00** | 0 0 | 白线 | 白线 | 停车 | 离线了，停下来 |

> [!TIP]
> 为什么是 10 和 01？因为代码里 `data2 * 10 + data3` 相当于用十进制模拟二进制拼接。`1*10+0 = 10`（左白右黑），`0*10+1 = 1`（左黑右白）。

### ③ 避障逻辑：左右红外 + 超声波

红外避障比超声波避障更灵敏——红外检测距离近（2-30cm），适合**近距离快速反应**：

```c
// 红外避障核心
if (right_led2 == 0)  // 右侧检测到障碍物
{
    left_run(120, 120);  // 向左转
}
if (left_led2 == 0)   // 左侧检测到障碍物
{
    right_run(120, 120); // 向右转
}
```

> [!INFO]
> 红外避障 + 超声波避障的区别：
> - **红外**：反应快、距离近（2-30cm），适合贴墙走、精细避障
> - **超声波**：反应慢、距离远（2-400cm），适合远距离预警

### ④ 完整循迹代码

```c
void Tracking(void)
{
    char data1, data2, data3;
    data2 = left_led1;       // 读左传感器
    data3 = right_led1;      // 读右传感器
    data1 = data2 * 10 + data3;  // 拼接状态

    if (data1 == 11) {       // 两线都在黑线上
        forward(120, 120);   // 前进
    }
    else if (data1 == 10) {  // 左白右黑 -> 偏左
        left_run(80, 160);   // 右转（差速转向）
    }
    else if (data1 == 1) {   // 左黑右白 -> 偏右
        right_run(160, 80);  // 左转（差速转向）
    }
    else {                   // 两线都白 -> 离线
        stop();              // 停车
    }
}
```

### ⑤ 四种模式一览

小车通过 KEY 按键切换 4 种模式：

| 模式 | LED 状态 | 功能 | 说明 |
| :---: | :---: | :--- | :--- |
| **0** | LED1灭, LED2灭 | 超声波避障 | 远距离预警 |
| **1** | LED1亮, LED2灭 | 红外避障+超声波 | 红外近距离+超声波远距离 |
| **2** | LED1灭, LED2亮 | 红外循迹+超声波 | 循迹途中遇障绕行 |
| **3** | LED1亮, LED2亮 | 纯红外循迹 | 沿黑线跑 |

## 1.3 实操注意事项

1. **调试电位器**：每个红外模块上都有电位器，先调至中间位置，再根据实际路面微调
2. **环境光影响**：强光（阳光、聚光灯）会干扰红外传感器，室内灯光影响较小
3. **黑线宽度**：建议黑线宽度 1.5-2cm，两条线间距 3-5cm，太窄或太宽都影响循迹效果
4. **检测距离**：红外模块离地面高度建议 1-2cm，太高灵敏度下降

### 任务二：红外避障模式（必做）

实现 `IRAvoid()` 函数：

- 右侧红外检测到障碍物 -> 左转

- 都没有 -> 前进

> [!TIP]
> 红外避障和超声波避障的区别：红外反应快但距离近，超声波反应慢但距离远。实际小车通常**两者配合使用**。

- 红外检测到侧方障碍物 -> 快速转向（贴墙绕行）

### 任务四：速度自适应（选做）

根据偏离程度调节速度：

| 偏离程度 | 左右轮速度差 | 说明 |
| :--- | :--- | :--- |
| 轻微偏离（1cm） | 差值 40 | 小幅度纠偏 |
| 明显偏离（3cm） | 差值 80 | 大幅度纠偏 |
| 完全离线 | 停车 | 重新寻找黑线 |

## 2.2 调试建议

- **传感器不工作**：检查供电是否为 5V，电位器是否调到合适位置
- **循迹抖动**：减小左右轮速度差，让转向更平滑
- **频繁误判**：在强光环境下红外会失效，尝试遮挡环境光或调低灵敏度
- **离线后不恢复**：在停车逻辑中加入"原地旋转搜索"
