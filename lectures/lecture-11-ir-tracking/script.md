# 一、前半部分（45 分钟）

## 1.1 红外循迹传感器原理

### ① 红外循迹的本质

红外循迹模块的工作原理：模块上的红外发射管持续发光，接收管检测反射回来的光强。**白色表面反射强，黑色表面吸收光**。模块内部的 LM393 比较器将反射光强与电位器设定的阈值比较，输出高/低电平。

| 地面颜色 | 反射强度 | 模块输出 | 含义 |
| :--- | :--- | :--- | :--- |
| **白色** | 强 | **低电平（0）** | 检测到白地 |
| **黑色** | 弱 | **高电平（1）** | 检测到黑线 |

> [!WARNING]
> **不同模块输出逻辑可能相反！** 有些模块是"白=1，黑=0"，接线前务必先验证：在白纸上放模块看输出，在黑纸上放模块看输出。

### ② 模块结构拆解

常见的红外循迹模块有 4 个引脚：

| 引脚 | 功能 | 说明 |
| :--- | :--- | :--- |
| VCC | 电源 | 3.3V-5V |
| GND | 地 | 共地 |
| DO | 数字输出 | 比较器输出，0 或 1 |
| AO | 模拟输出 | 反射强度电压（本实验用不到） |

核心是 **LM393 比较器**：红外发射管发光 → 接收管收到反射光 → 产生电压 → 与电位器设定的阈值比较 → 输出高/低。

> [!TIP]
> **电位器的作用**：模块上那个蓝色小旋钮用来调节检测阈值。调好之后，白地上输出一种电平，黑地上输出另一种电平，中间过渡带越小越好。

### ③ 小车上的循迹传感器

小车通常安装 **2 个**循迹模块，安装在车底朝下：

| 传感器 | 引脚 | 说明 |
| :--- | :--- | :--- |
| 左循迹 | P3^5 | 检测左侧黑线 |
| 右循迹 | P3^4 | 检测右侧黑线 |

> [!NOTE]
> 引脚名 `left_led1` / `right_led1` 是历史遗留命名，实际功能是红外传感器，不是 LED。`led1` 系列用于循迹，`led2` 系列用于避障。

### ④ 实操必看的关键细节

讲完原理，补充几个实操时必须注意的点：

1. **调电位器**：每个红外模块上的电位器先调至中间位置，再根据实际路面微调。在白地上输出 0，在黑地上输出 1 即可。
2. **环境光影响**：强光（阳光、聚光灯）会干扰红外传感器，室内灯光影响较小。建议在室内灯光下调试，避免阳光直射。
3. **黑线宽度**：建议 1.5-2cm，两条线间距 3-5cm。线太窄传感器容易扫过，太宽会延迟响应。
4. **安装高度**：红外模块离地面高度建议 1-2cm，太高灵敏度下降。安装时注意保持左右水平。

## 1.2 循迹逻辑

### 核心思路：4 种状态

把左右传感器的读数拼成一个数，根据这个数决定动作：

```c
// 把两个传感器的状态拼成一个两位数
char data1, data2, data3;
data2 = left_led1;      // 左：0=白，1=黑
data3 = right_led1;     // 右：0=白，1=黑
data1 = data2 * 10 + data3;  // 拼成：11/10/01/00
```

| data1 值 | 左传感器 | 右传感器 | 小车动作 | 含义 |
| :--- | :--- | :--- | :--- | :--- |
| **11** | 黑 | 黑 | 前进 | 两线在线中间，正常走 |
| **10** | 白 | 黑 | **右转** | 偏左了，往右纠 |
| **01** | 黑 | 白 | **左转** | 偏右了，往左纠 |
| **00** | 白 | 白 | **停车** | 离线了，停下来 |

> [!TIP]
> **为什么差速转向？** 左右轮速度不相等，产生差速让小车转弯。速度差越大，转弯越急。

### 状态机理解

把循迹看作一个**有限状态机**：

```
         进入循环
            |
     +------+------+
     |             |
  左=黑 右=黑   左=白 右=黑
  (11)        (10)
     |             |
   前进          右转
     |             |
     +------+------+
            |
     +------+------+
     |             |
  左=黑 右=白   左=白 右=白
  (01)        (00)
     |             |
   左转          停车
```

> [!NOTE]
> 用 `data2 * 10 + data3` 而不是 `data2 * 2 + data3` 的原因：
> - `* 10` 得到的是十进制数（11、10、1、0），对应 if-else 的判断直观
> - `* 2` 得到的是二进制拼接（11=3、10=2、01=1、00=0），用 switch-case 更合适
> - 两种方法都可以，看个人习惯

### 循迹 vs 避障

| | 红外循迹 | 红外避障 |
| :--- | :--- | :--- |
| **传感器位置** | 车底朝下 | 车身两侧 |
| **检测目标** | 地面黑线 | 侧方障碍物 |
| **输出逻辑** | 白=0，黑=1 | 通常：无障碍=1，有障=0 |
| **转向方式** | 差速转向（纠偏） | 等速转向（快速避让） |
| **状态数** | 4 种（11/10/01/00） | 3 种（右有障/左有障/都无障碍） |

## 1.3 代码实现

上节课学了红外避障，这节课在避障基础上进阶——**从"避开障碍物"到"沿着黑线走"**。

### 第 1 步：引脚定义

引脚定义在 [config.h](code/config.h) 中：

```c
// config.h
sbit left_led1   = P3^5;   // 左循迹传感器（白=0，黑=1）
sbit right_led1  = P3^4;   // 右循迹传感器（白=0，黑=1）
```

> [!TIP]
> 循迹传感器接在 P3 口，不占用 P1（电机 PWM）和 P2（超声波），互不冲突。

### 第 2 步：读传感器

```c
// 读传感器状态
char data2 = left_led1;    // 读左传感器
char data3 = right_led1;   // 读右传感器
```

### 第 3 步：拼接 + 判断

```c
// 拼接状态
char data1 = data2 * 10 + data3;

// 根据状态决定动作
if (data1 == 11) {
    forward(120, 120);     // 两线在线 → 前进
}
else if (data1 == 10) {
    left_run(80, 160);     // 左白右黑 → 右转纠偏
}
else if (data1 == 1) {
    right_run(160, 80);    // 左黑右白 → 左转纠偏
}
else {
    stop();                // 两线都离线 → 停车
}
```

## 2.1 完整代码

### 基础版：循迹函数

在 [main.c](code/main.c) 中，完善 `Tracking()` 函数：

```c
void Tracking(void)
{
    // 1. 读取左右传感器
    char data2 = left_led1;    // 左：0=白，1=黑
    char data3 = right_led1;   // 右：0=白，1=黑

    // 2. 拼接状态
    char data1 = data2 * 10 + data3;

    // 3. 根据状态决定动作
    if (data1 == 11) {         // 两线在线 → 前进
        forward(120, 120);
    }
    else if (data1 == 10) {    // 左白右黑 → 右转纠偏
        left_run(80, 160);
    }
    else if (data1 == 1) {     // 左黑右白 → 左转纠偏
        right_run(160, 80);
    }
    else {                     // 两线都离线 → 停车
        stop();
    }
}
```

### 进阶版：速度自适应循迹

根据偏离程度调节速度差，让循迹更平滑：

| 偏离程度 | 左右轮速度差 | 说明 |
| :--- | :--- | :--- |
| 轻微偏离（data1=10 或 01） | 差值 40 | 小幅度纠偏 |
| 明显偏离（接近离线） | 差值 80 | 大幅度纠偏 |
| 完全离线（data1=00） | 停车 | 重新寻找黑线 |

```c
void Tracking_Adaptive(void)
{
    char data2 = left_led1;
    char data3 = right_led1;
    char data1 = data2 * 10 + data3;

    if (data1 == 11) {
        forward(120, 120);           // 两线在线 → 前进
    }
    else if (data1 == 10) {
        left_run(100, 140);          // 轻微偏离 → 小幅度纠偏
    }
    else if (data1 == 1) {
        right_run(140, 100);         // 轻微偏离 → 小幅度纠偏
    }
    else {
        stop();                      // 完全离线 → 停车
    }
}
```

### 进阶版：原地旋转搜索

当两传感器都离线（data1 == 0）时，不要直接停车，改为原地旋转搜索黑线：

```c
void Tracking_Search(void)
{
    char data2 = left_led1;
    char data3 = right_led1;
    char data1 = data2 * 10 + data3;

    if (data1 == 11) {
        forward(120, 120);
    }
    else if (data1 == 10) {
        left_run(80, 160);
    }
    else if (data1 == 1) {
        right_run(160, 80);
    }
    else {                         // 两线都离线 → 原地旋转搜索
        left_run(100, 100);        // 原地旋转
        Delay_Ms(200);
    }
}
```

### 进阶版：红外循迹 + 红外避障组合

在循迹基础上叠加避障功能，遇到障碍物优先避障：

```c
void Car_Control(void)
{
    // 1. 先检查避障（优先级最高）
    if (left_led2 == 0 || right_led2 == 0) {
        // 有障碍物，执行避障
        if (right_led2 == 0) {
            left_run(120, 120);    // 右侧有障 → 左转
        }
        else {
            right_run(120, 120);   // 左侧有障 → 右转
        }
        return;
    }

    // 2. 无障碍，执行循迹
    char data2 = left_led1;
    char data3 = right_led1;
    char data1 = data2 * 10 + data3;

    if (data1 == 11) {
        forward(120, 120);
    }
    else if (data1 == 10) {
        left_run(80, 160);
    }
    else if (data1 == 1) {
        right_run(160, 80);
    }
    else {
        stop();
    }
}
```

## 2.2 调试建议

- **传感器不识别黑线**：调电位器直到模块输出稳定切换。在白地上 LED 应该灭，在黑线上 LED 应该亮。
- **循迹抖动（左右摇摆）**：减小转向时的速度差。如果还是抖，在循环末尾加 `Delay_Ms(10~20)` 降低采样频率。
- **小车跑偏**：检查左右轮安装是否水平，红外模块安装高度是否在 1-2cm。
- **直线走不直**：先让小车在白色地面上前进，微调左右轮 PWM 占空比使两轮速度一致，再开始循迹。
