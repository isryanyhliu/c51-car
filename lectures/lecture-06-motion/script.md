### ② 资料-引脚定义
手册原文：  
![](assets/Pasted%20image%2020260426171055.png)

**翻译 + 大白话：**
- **ENA = 左电机使能（对应你代码：Left_moto_pwm）**
- **ENB = 右电机使能（对应你代码：Right_moto_pwm）**
- **ENA = 低电平（0）→ 左电机直接禁用、停转**
- **ENA = 高电平（1）→ 左电机允许工作**
- ENB 同理

L298 驱动电机分 **两个完全独立的控制**：

|引脚|名字|作用|控制内容|
|---|---|---|---|
|IN1 / IN2|方向引脚|控制电机**正转 / 反转 / 停止**|方向|
|ENA|使能 / 调速引脚|控制电机**开 / 关 + 速度**|转速|

**PWM 调速**
- 给 ENA 一直输 **高电平 1** → 电机**全速转**
- 给 ENA 一直输 **低电平 0** → 电机**停止**
- 给 ENA **快速切换高低电平（PWM）** → 电机**调速**（高电平时间越长，速度越快）

**IN1/IN2 管方向，ENA 管速度和开关****
**写驱动代码 = 看芯片手册 + 看板子丝印，自己定义头文件**


### ③ 引脚+丝印：头文件定义

```c
// 完全来自：L298手册 + 小车硬件丝印 
sbit Left_moto_pwm = P1^5; // ENA 左电机使能+调速 
sbit IN1 = P1^4; // 左方向 
sbit IN2 = P1^3; // 左方向 
sbit Right_moto_pwm = P1^0; // ENB 右电机使能+调速 
sbit IN3 = P1^2; // 右方向 
sbit IN4 = P1^1; // 右方向
```

  
>[!IMPORTANT]  
>所以，头文件相当于说明书  
  
  
>[!CAUTION]  
>为什么硬件生产商不直接把头文件给我们，而是给一个操作手册而已


**芯片厂商只生产「通用硬件」，不生产「你的板子」；

接线是开发板厂家自定义的，没有统一标准，所以不可能给你现成头文件。

1.芯片原厂（ST、Atmel、乐鑫等，比如 L298、51 内核）只负责出**数据手册 (Datasheet)**：
- 芯片电气规则
- 引脚功能定义（ENA 是使能、IN1 是方向）
- 真值表、电平逻辑、时序
**绝对不能写死引脚**

比如 L298 的 ENA：
- 你的小车：ENA → P1^5
- 别家开发板：ENA → P2^0
- 竞赛板子：ENA → P3^1

如果原厂写死头文件 `sbit ENA = P1^5;`换个板子直接报废，芯片就没法通用售卖。


**开发板 / 小车小厂（做你这块 QX_A11 板子的厂家）**

他们**明明可以给完整头文件 + 全套代码**，但为什么不给？

1. **硬件迭代极快**
    改一版板子、换个走线、换引脚，旧头文件直接作废。
    手册不用改，头文件要全改，维护麻烦。
2. **故意留白，留给二次开发**
    全都给你写死了，还怎么上课、怎么让学生练引脚配置？
    

**头文件本来就该开发者自己写**，这是嵌入式全世界统一的规矩：
- 芯片厂：给**功能规则**（手册）【**引脚功能**】
- 板子厂：给**丝印接线**（板上印 P16、P15、ENA）【**引脚位置**】
- **开发者：自己结合两者，手写引脚头文件**

### ④ 软件PWM vs 定时器PWM

 **一句话核心区别**

✅ **软件 PWM**：**CPU 亲自盯着**，不停开关引脚，**啥也不能干，专门生成 PWM**

✅ **定时器 PWM**：**硬件定时器自动干活**，CPU 该干嘛干嘛，**完全不耽误主程序**


**工作原理**

1. 软件 PWM（你现在用的）

- 靠 `Delay_us` 延时函数**手动控制**ENA/ENB 引脚高低电平
- **CPU 全程阻塞**，执行 PWM 的时候，不能执行别的代码
- 代码简单，纯靠循环延时

2. 定时器 PWM（工业 / 比赛标准用法）

- 用 51 内部**硬件定时器**自动计时，时间到了自动翻转引脚
- **CPU 完全解放**，主程序可以同时跑按键、LED、传感器
- 精度高、不受主程序延时影响

**对比表格**

|对比项|软件 PWM|定时器 PWM|
|---|---|---|
|**谁来执行**|CPU 软件延时|硬件定时器 自动执行|
|**占用 CPU**|占满 CPU，不能干别的事|0 占用，CPU 自由运行|
|**精度**|低，受主程序延时干扰|极高，稳定无误差|
|**代码难度**|极低，新手秒懂|中等，要懂定时器中断|
|**多任务**|不能同时控制电机 + 读按键 / 传感器|可以同时跑所有功能|
|**小车表现**|调速时会卡顿、偶尔抽搐|平稳顺滑，无卡顿|
|**适用场景**|教学入门、演示原理|正式运行、循迹 / 避障小车|
