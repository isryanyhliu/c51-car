## 1.1 超声波传感器原理

![](../../assets/images/Pasted%20image%2020260505143838.png)

超声波传感器是一种利用超声波进行测距的传感器，主要用于非接触式距离测量。它的工作原理基于声波的反射：传感器发射超声波信号，当信号遇到障碍物时会反射回来，传感器接收到回波信号，通过计算发射和接收的时间差来计算距离。

### ① 从 "山谷回声" 说起：先懂底层逻辑

同学们，我们先从一个大家都熟悉的生活场景引入：

如果你站在空旷的山谷里，对着远处的山崖喊一声，过几秒会听到回声。
这个过程里藏着一个朴素的物理原理：

- 你 "喊" 出的声音，会以固定速度在空气中传播；
- 遇到山崖反射回来，被你的耳朵 "听" 到；
- 只要算出 "喊出去" 到 "听到回声" 的时间差，再乘上声音的速度
  就能算出你和山崖的距离。

我们今天要讲的**HC-SR04 超声波模块**，本质上就是一个会主动 "喊"、也会精准 "听回声" 的模块。它用的就是这个 "回声测距" 的原理，只是把 "人喊的声音" 换成了方向性更好、更稳定的超声波。

### ② HC-SR04 是怎么 "喊" 和 "听" 的？—— 工作流程拆解

我们把模块的工作过程拆成 4 步，每一步都和刚才的山谷回声对应，大家就很容易理解了：

1. 给模块一个 "启动指令"：
   1. 我们需要给模块的`Trig`（触发）引脚，发一个**至少 10 微秒的高电平脉冲**
   2. 这就相当于给模块发一个指令："准备好，开始'喊'了！"
2. 模块自动 "喊" 出超声波：
   1. 收到指令后，模块会自动向外发射**8 个 40kHz 的超声波脉冲**。
   2. 这里补充一个小细节：为什么用 40kHz？因为这个频率的超声波方向性好，不容易绕开障碍物，也不容易被空气吸收，刚好适合用来做近距离测距，这也是模块测距稳定的关键。
3. 模块 "听" 回波信号
   1. 这些超声波向前传播，遇到障碍物后会反射回来，模块上的接收探头会 "捕捉" 到这个回波信号。
4. 模块输出 "时间差" 信号
   1. 一旦接收到回波，模块的`Echo`（回响）引脚就会输出一个**高电平信号**
   2. 这个高电平持续的时间，正好就是超声波从 "发射" 到 "返回" 的总时间。就相当于我们 "喊出去" 到 "听到回声" 的时间差。

### ③ 距离怎么算？—— 从基础公式到简化公式

有了 "往返时间差"，我们就能算出距离了。
这里我们从物理公式推起，让大家明白代码里的公式不是凭空来的：

1. 基础物理公式
   1. 声音在空气中的传播速度约为 **340m/s**，而超声波从模块到障碍物再返回，走的是一个 "来回"，所以实际距离是总路程的一半：

$$
\text{距离} = \frac{\text{声速} \times \text{往返时间}}{2}
$$

2. 适配单片机的简化公式
   1. 在单片机里，我们读到的时间单位通常是**微秒（uS）**
   2. 我们可以把公式换算一下，让计算更方便：
   - 先把声速换算成 "厘米 / 微秒"：
     $340\ \text{m/s} = 34000\ \text{cm/s} = 0.034\ \text{cm/uS}$（因为 1 秒 = 1,000,000 微秒）
   - 代入基础公式：
     $\text{距离(cm)} = \frac{0.034\ \text{cm/uS} \times \text{时间(uS)}}{2} = \text{时间(uS)} \times 0.017\ \text{cm/uS}$
   - 换算成毫米：
     $\text{距离(mm)} = \text{时间(uS)} \times 0.17$

3. 代码中的最终公式

实际代码中公式为：

```c
Distance = (float)(Time * 1.085) * 0.17;
```

这里 `Time` 是定时器的计数值，两个系数的来源：

- **`1.085`**：晶振补偿系数。定时器基于 11.0592MHz 晶振，1 个机器周期 = 12 / 11.0592 ≈ 1.085μs（标准 12MHz 晶振的系数是 1.0）。定时器每计一个数，实际过了 1.085μs，所以要乘这个系数换算真实时间。
- **`0.17`**：声速换算系数。每微秒 0.17mm（由 340m/s 换算而来）。

### ④ 实操必看的关键细节（结合大家容易踩的坑）

讲完原理，我们补充几个实操时必须注意的点，这些细节直接影响模块能不能正常工作：

1. **两次测量间隔要大于 60ms**：
   有两个原因：
   - **模块恢复时间**：超声波的回波可能有延迟，两次测量间隔太短，上一次的回波还没消失，就会干扰下一次的测量。
   - **定时器溢出保护**：Timer1 是 16 位定时器，最大计数 65535，对应约 65ms（基于 11.0592MHz 晶振）。如果间隔超过 65ms，定时器可能已经溢出，读到的时间值就不对了。60ms 是安全余量。

2. **带电连接时，必须先接 GND**：
   这个模块对电源波动比较敏感，如果带电接线时先接了信号线，模块电平不稳，很容易烧坏。所以大家接线时，一定要先把地线接好，再接其他线。

3. **被测物体要 "够大、够平整"**：
   如果物体太小、表面太粗糙，超声波会被散射掉，模块就 "听不到" 回波了，测出来的数据会飘。一般来说，被测物体的面积最好不小于 0.5 平方米，而且尽量是平面，这样回波信号才够稳定。

## 1.2 代码实现

上节课我们用**定时器**实现了电机 PWM 调速；

本节课我们还是用**定时器**，只不过不再控制电机，而是**测量超声波的往返时间**，最终算出距离

### ① 硬件接线（动手第一步，零门槛）

HC-SR04 只有 **4 个引脚**，对应 51 单片机接线如下：

| HC-SR04 引脚 | 功能     | 51 单片机接线 | 必须注意 |
| :----------- | :------- | :------------ | :------- |
| VCC          | 电源正极 | 5V            | 必须 5V，不能接 3.3V |
| GND          | 电源负极 | GND           | 优先接 GND，防止烧模块 |
| Trig（T）    | 触发发射 | P2^1          | 输出引脚 |
| Echo（E）    | 接收回响 | P2^0          | 输入引脚 |

> [!NOTE]
> 以上引脚定义以本项目 [config.h](code/config.h) 为准，实际接线时请对照自己的电路设计。

### ② 代码引脚定义

引脚定义在项目文件 [config.h](code/config.h) 中：

```c
// config.h
sbit TrigPin = P2^1;   // 触发引脚
sbit EchoPin = P2^0;   // 回响引脚
```

> [!TIP]
> - `TrigPin`：单片机给超声波发**启动指令**的脚
> - `EchoPin`：超声波给单片机**回传时间**的脚

### ③ 核心功能拆解

**函数 1：触发超声波发射（[ultrasonic.c](code/ultrasonic.c)）**

```c
// 作用：给 Trig 脚发一个 10 微秒的高电平，让超声波开始 "喊"
void TrigUltrasonic(void)
{
    TrigPin = 0;       // 先拉低，复位
    _nop_();           // 空操作，延时约 1us
    TrigPin = 1;       // 拉高，启动发射
    // 保持 10us 高电平（每个 _nop_() 约 1us）
    _nop_();_nop_();_nop_();_nop_();_nop_();
    _nop_();_nop_();_nop_();_nop_();_nop_();
    TrigPin = 0;       // 拉低，结束触发
}
```

> [!INFO]
> 就像按开关 —— **先关 -> 再开 10 微秒 -> 再关**
> 超声波收到这个信号，就会自动发射 8 个 40kHz 超声波。
>
> > [!WARNING]
> > 高电平期间的 `_nop_()` **不能省略**！如果直接拉高再拉低，脉冲宽度不够 10μs，模块可能不响应。

**函数 2：获取距离（[ultrasonic.c](code/ultrasonic.c)）**

```c
// 作用：用定时器 1 测回声时间 -> 计算距离
unsigned int Get_Distance(void)
{
    unsigned int Distance, Time;

    TrigUltrasonic();              // 第一步：触发超声波

    while(!EchoPin);               // 等待回声开始（Echo 变高）
    TR1 = 1;                       // 第二步：启动定时器 1（开始计时）
    while(EchoPin);                // 等待回声结束（Echo 变低）
    TR1 = 0;                       // 第三步：关闭定时器 1（停止计时）

    Time = TH1 * 256 + TL1;        // 读取定时器总时间
    TH1 = 0; TL1 = 0;              // 清空定时器，下次用

    // 计算距离（单位：毫米）
    Distance = (float)(Time * 1.085) * 0.17;
    return Distance;
}
```

> [!TIP]
> 1.  **触发**：让超声波发射信号
> 2.  **计时**：Echo 变高 -> 开定时器，Echo 变低 -> 关定时器（时间 = 声波往返时间）
> 3.  **计算**：用时间算距离（公式见上文解释）

> [!NOTE]
> `while(!EchoPin)` 和 `while(EchoPin)` 在**没有回波**的情况下会一直等待。实际产品中需要加超时保护，我们代码里用了 Timer1 中断兜底：当 Timer1 溢出（约 65ms）时，中断服务函数会自动把 `EchoPin` 拉低，`while(EchoPin)` 就会退出。所以教学代码不会死循环，但读到的距离值会是 65ms 对应的最大值，后续判断会把它当作"无障碍物"。

### ⑤ 进阶：超声波 + 电机避障

**基础版避障逻辑**

1. 测距离
2. **距离 < 300mm** -> 停车
3. **距离 > 300mm** -> 前进

```c
// main.c 中的避障逻辑
if(dis > OBSTACLE_LIMIT)  // 无障碍物（> 300mm）
{
    forward();            // 小车前进
}
else                     // 检测到障碍物（<= 300mm）
{
    stop();               // 停车
}
```

**进阶版：避障 + 转向（推荐课后实现）**

```c
if(dis > 300) {
    forward();                    // 无障碍，前进
}
else {
    stop();
    Delay_Ms(100);
    back_run();                   // 后退，脱离当前位置
    Delay_Ms(300);
    stop();
    left_run();                   // 左转，寻找新方向
    Delay_Ms(200);
    stop();
}
```

> [!TIP]
> 进阶版避障增加了"后退 + 转向"动作，避免原地死等。可以进一步加左右转向选择（先左探，再右探，选空的一侧走）。

| :------- | :--- |
| > 300mm | 全速前进 |
| 150-300mm | 减速前进（降低 PWM 占空比） |
| < 150mm | 紧急避障（后退 + 转向） |

## 任务三：多次采样滤波（选做）

超声波读数会有抖动，单次采样容易误判。实现**5 次采样取平均**：

```c
unsigned int Get_Distance_Filt(void)
{
    unsigned int sum = 0;
    for (int i = 0; i < 5; i++) {
        sum += Get_Distance();
        Delay_Ms(60);
    }
    return sum / 5;
}
```

## 调试建议

- **读不到回波**：检查接线是否正确（Trig/P2^1, Echo/P2^0）、供电是否为 5V、被测物体是否平整
- **距离数值跳变**：尝试加滤波（任务三）
- **模块不工作**：先单独烧录测距代码验证模块，再接入避障逻辑
