## 1.1 超声波传感器原理

![](../../assets/images/Pasted%20image%2020260505143838.png)

超声波传感器是一种利用超声波进行测距的传感器，主要用于非接触式距离测量。它的工作原理基于声波的反射：传感器发射超声波信号，当信号遇到障碍物时会反射回来，传感器接收到回波信号，通过计算发射和接收的时间差来计算距离。

### ① 从 “山谷回声” 说起：先懂底层逻辑

同学们，我们先从一个大家都熟悉的生活场景引入：

如果你站在空旷的山谷里，对着远处的山崖喊一声，过几秒会听到回声。  
这个过程里藏着一个朴素的物理原理：

- 你 “喊” 出的声音，会以固定速度在空气中传播；
- 遇到山崖反射回来，被你的耳朵 “听” 到；
- 只要算出 “喊出去” 到 “听到回声” 的时间差，再乘上声音的速度  
  就能算出你和山崖的距离。

我们今天要讲的**HC-SR04 超声波模块**，本质上就是一个会主动 “喊”、也会精准 “听回声” 的模块。它用的就是这个 “回声测距” 的原理，只是把 “人喊的声音” 换成了方向性更好、更稳定的超声波。

### ② HC-SR04 是怎么 “喊” 和 “听” 的？—— 工作流程拆解

我们把模块的工作过程拆成 4 步，每一步都和刚才的山谷回声对应，大家就很容易理解了：

1. 给模块一个 “启动指令”：
    1. 我们需要给模块的`Trig`（触发）引脚，发一个**至少 10 微秒的高电平脉冲**
    2. 这就相当于给模块发一个指令：“准备好，开始‘喊’了！”
2. 模块自动 “喊” 出超声波：
    1. 收到指令后，模块会自动向外发射**8 个 40kHz 的超声波脉冲**。
    2. 这里补充一个小细节：为什么用 40kHz？因为这个频率的超声波方向性好，不容易绕开障碍物，也不容易被空气吸收，刚好适合用来做近距离测距，这也是模块测距稳定的关键。
3. 模块 “听” 回波信号
    1. 这些超声波向前传播，遇到障碍物后会反射回来，模块上的接收探头会 “捕捉” 到这个回波信号。
4. 模块输出 “时间差” 信号
    1. 一旦接收到回波，模块的`Echo`（回响）引脚就会输出一个**高电平信号**
    2. 这个高电平持续的时间，正好就是超声波从 “发射” 到 “返回” 的总时间。就相当于我们 “喊出去” 到 “听到回声” 的时间差。

### ③ 距离怎么算？—— 从基础公式到简化公式

有了 “往返时间差”，我们就能算出距离了。  
这里我们从物理公式推起，让大家明白代码里的公式不是凭空来的：

1. 基础物理公式
    1. 声音在空气中的传播速度约为 **340m/s**，而超声波从模块到障碍物再返回，走的是一个 “来回”，所以实际距离是总路程的一半：

$$
\text{距离} = \frac{\text{声速} \times \text{往返时间}}{2}
$$

2. 适配单片机的简化公式
    1. 在单片机里，我们读到的时间单位通常是**微秒（uS）**
    2. 我们可以把公式换算一下，让计算更方便：
    - 先把声速换算成 “厘米 / 微秒”：  
      $340\ \text{m/s} = 34000\ \text{cm/s} = 0.034\ \text{cm/uS}$（因为 1 秒 = 1,000,000 微秒）
    - 代入基础公式：  
      $\text{距离(cm)} = \frac{0.034\ \text{cm/uS} \times \text{时间(uS)}}{2} = \text{时间(uS)} \times 0.017\ \text{cm/uS}$

### ④ 实操必看的关键细节（结合大家容易踩的坑）

讲完原理，我们补充几个实操时必须注意的点，这些细节直接影响模块能不能正常工作：

1. **两次测量间隔要大于 60ms**：  
   超声波的回波可能会有延迟，如果两次测量间隔太短，上一次的回波还没消失，就会干扰下一次的测量，导致数据不准。所以代码里，我们每次测距后，最好让模块 “休息” 60ms 以上再测下一次。
2. **带电连接时，必须先接 GND**：  
   这个模块对电源波动比较敏感，如果带电接线时先接了信号线，模块电平不稳，很容易烧坏。所以大家接线时，一定要先把地线接好，再接其他线。
3. **被测物体要 “够大、够平整”**：  
   如果物体太小、表面太粗糙，超声波会被散射掉，模块就 “听不到” 回波了，测出来的数据会飘。一般来说，被测物体的面积最好不小于 0.5 平方米，而且尽量是平面，这样回波信号才够稳定。

## 1.2 代码实现

上节课我们用**定时器**实现了电机 PWM 调速；

本节课我们还是用**定时器**，只不过不再控制电机，而是**测量超声波的往返时间**，最终算出距离

### ① 硬件接线（动手第一步，零门槛）

HC-SR04 只有 **4 个引脚**，对应 51 单片机接线如下：

| HC-SR04 引脚 | 功能     | 51 单片机接线               | 必须注意               |
| :----------- | :------- | :-------------------------- | :--------------------- |
| VCC          | 电源正极 | 5V                          | 必须 5V，不能接 3.3V   |
| GND          | 电源负极 | GND                         | 优先接 GND，防止烧模块 |
| Trig（T）    | 触发发射 | TrigPin（源码定义的 IO 口） | 输出引脚               |
| Echo（E）    | 接收回响 | EchoPin（源码定义的 IO 口） | 输入引脚               |

✅ 课堂口诀：**5V 供电、GND 共地、Trig 发信号、Echo 听回声**

### ② 代码引脚定义（告诉单片机：哪个脚接超声波）

```c
// 这两个定义在 QX_A11.h 里，对应超声波的两个控制脚
sbit TrigPin = P1^0;  // 触发引脚：P1.0
sbit EchoPin = P1^1;  // 回响引脚：P1.1
```

✅ 一句话理解：

- `TrigPin`：单片机给超声波发**启动指令**的脚
- `EchoPin`：超声波给单片机**回传时间**的脚

### ③ 核心功能拆解（只讲超声波，去掉复杂电机 / 模式）

**函数 1：触发超声波发射（QXMBOT_TrigUltrasonic）**

```c
// 作用：给 Trig 脚发一个**10 微秒的高电平**，让超声波开始 “喊”
void QXMBOT_TrigUltrasonic()
{
	TrigPin = 0; // 先拉低，复位
	_nop_();     // 空操作，延时1us
	TrigPin = 1; // 拉高，启动发射
	// 保持10us高电平（源码用多个空操作实现）
	TrigPin = 0; // 拉低，结束触发
}
```

> [!INFO]
> 就像按开关 ——**先关→再开 10 微秒→再关**  
> 超声波收到这个信号，就会自动发射 8 个 40kHz 超声波。

**函数 2：获取距离（QXMBOT_GetDistance）**

```c
// 作用：用**定时器 1**测回声时间 → 计算距离
unsigned int QXMBOT_GetDistance()
{
	unsigned int Distance;  // 存放最终距离
	unsigned int Time;      // 存放定时器测的时间

	QXMBOT_TrigUltrasonic();// 第一步：触发超声波

	while(!EchoPin);   // 等待回声开始（Echo变高）
	Timer1On;			      // 第二步：启动定时器1（开始计时）
	while(EchoPin);		  // 等待回声结束（Echo变低）
	Timer1Off;			    // 第三步：关闭定时器1（停止计时）

	Time = TH1*256 + TL1;  // 读取定时器的总时间
	TH1=0; TL1=0;          // 清空定时器，下次用

	// 源码公式：计算距离（单位：毫米）
	Distance = (float)(Time*1.085)*0.17;
	return Distance;
}
```

> [!TIP]
>
> 1.  **触发**：让超声波发射信号
> 2.  **计时**：Echo 变高→开定时器，Echo 变低→关定时器（时间 = 声波往返时间）
> 3.  **计算**：用时间算距离（源码已经写好公式，直接用）

### ④ 极简程序（纯超声波测距，可直接烧录）

```c

#include <reg52.h>
#include <intrins.h>

// 1. 定义超声波引脚
sbit TrigPin = P1^0;
sbit EchoPin = P1^1;

// 2. 定时器1初始化（测时间用，上节课学过）
void Timer1_Init(void)
{
	TMOD |= 0x10;	// 16位定时模式
	TH1 = 0;
  TL1 = 0;
}

// 3. 触发超声波
void QXMBOT_TrigUltrasonic()
{
	TrigPin = 0;
	_nop_();_nop_();
	TrigPin = 1;
	_nop_();_nop_();_nop_();_nop_();_nop_();
	TrigPin = 0;
}

// 4. 获取距离
unsigned int QXMBOT_GetDistance()
{
	unsigned int Distance,Time;
	QXMBOT_TrigUltrasonic();
	while(!EchoPin);
	TR1 = 1; // 启动定时器1
	while(EchoPin);
	TR1 = 0; // 关闭定时器1
	Time=TH1*256+TL1;
	TH1=0;TL1=0;
	Distance = (float)(Time*1.085)*0.17;
	return Distance;
}

// 主函数：循环测距
void main(void)
{
	Timer1_Init();
	while(1)
	{
		unsigned int dis = QXMBOT_GetDistance(); // 读取距离（毫米）
		// 这里可以加串口/数码管显示距离
	}
}
```

### ⑤ 进阶：超声波 + 电机避障（对应你的源码）

**避障逻辑（示例）**

1. 测距离
2. **距离 < 设定值** → 停车→后退→左转
3. **距离 > 设定值** → 前进

```c
// 你的源码避障函数（简化讲解版）
void UltrasonicAvoid(unsigned int 阈值)
{
	unsigned int Dis = QXMBOT_GetDistance(); // 测距离

	if(Dis < 阈值)  // 障碍物太近
	{
		stop();      // 停车
		back_run();  // 后退
		left_run();  // 左转
	}
	else             // 没有障碍物
	{
		forward();  // 前进
	}
}
```

> [!WARNING]
>
> 1.  超声波核心：**回声测距**，用定时器测往返时间
> 2.  两个引脚：`Trig`（发指令）、`Echo`（传时间）
> 3.  代码逻辑：**触发→计时→计算**
> 4.  避障逻辑：**近停远走**
