# 第 10 课：红外避障

---

# 课堂引入

> 上节课（第 8-9 课）我们学了超声波避障——小车能"看"到前方几米远的障碍物。但超声波有两个局限：一是反应慢（发射超声波 + 等待回波需要几毫秒），二是对软质物体（如布料、地毯）几乎无效。
>
> 实际的小车还需要一种**更快、更灵敏**的传感器来处理近距离障碍——这就是红外避障模块要解决的问题。

---

# 一、前半部分（45 分钟）

## 1.1 红外避障传感器原理
![](../../Pasted%20image%2020260517094823.png)

### ① 红外避障的本质

> **回顾**：第 8-9 课我们学了超声波避障——通过发射超声波、等待回波来测距。但超声波反应慢（需要几毫秒等待回波），而且对软质物体（如布料）无效。这节课我们换一个更快的方案：**红外避障**，直接读电平，反应更快。

红外避障模块的工作原理其实很直观：模块上的红外发射管持续发光，接收管检测反射回来的光强。当前方有障碍物时，反射光变强；无障碍时，反射光弱。

> [!WARNING]
> **不同模块输出逻辑可能相反！** 有些模块是"有障=0"，有些是"有障=1"。**没有统一标准**，接线前务必先验证：用手靠近传感器，看模块上 LED 的亮灭变化，确认你手头模块的逻辑。

### ② 模块结构拆解

常见的红外避障模块有 3 个引脚：

| 引脚 | 功能 | 说明 |
| :--- | :--- | :--- |
| VCC | 电源 | 3.3V-5V |
| GND | 地 | 共地 |
| OUT | 数字输出 | 比较器输出，0 或 1（接单片机） |

红外发射管发光 → 接收管收到反射光 → 产生电压 → 与电位器设定的阈值比较 → 输出高/低。

> [!TIP]
> **电位器的作用**：模块上那个蓝色小旋钮用来调节检测距离。顺时针调：灵敏度降低（检测距离变近）；逆时针调：灵敏度提高（检测距离变远）。

### ③ 小车上的避障传感器

小车通常安装 **2 个**避障模块，分别检测左右两侧：

| 传感器 | 引脚 | 说明 |
| :--- | :--- | :--- |
| 左避障 | P3^6 | 检测左侧障碍物 |
| 右避障 | P3^7 | 检测右侧障碍物 |


### ④ 实操必看的关键细节

讲完原理，补充几个实操时必须注意的点：

> [!IMPORTANT]
> **先测再写！** 动手写代码前，必须先完成以下验证：
> 1. 用手靠近每个传感器，记录模块 LED 的亮/灭变化
> 2. 确认你的模块是"有障=0"还是"有障=1"
> 3. 记下结果，写代码时对照你的实测值，不要想当然
>
> 不同批次、不同商家的模块输出逻辑可能完全相反，**实测是唯一标准**。

1. **调电位器**：避障检测距离建议调到 **5-15cm**。太远会提前转向，太近会撞上。
2. **侧面安装角度**：避障模块应朝**侧前方 30-45 度**安装，不要正对侧面，否则反应太晚。
3. **障碍物颜色**：黑色/吸光物体反射弱，检测距离会缩短。白色/反光物体检测距离更远。

## 1.2 避障逻辑

### 核心思路：4 种状态

把左右传感器的读数分开判断，根据状态决定动作：

```c
// 读避障传感器（示例代码，实际需根据场地调整）
char left_led  = left_ir;      // 左：0=有障，1=无障碍
char right_led = right_ir;     // 右：0=有障，1=无障碍

// 根据状态决定动作（示例逻辑，实际需根据场地调整）
if (right_led == 0 && left_led == 0) {
    // 两侧都有障 -> 后退 + 转向
    back_run(120, 120);
    Delay_Ms(300);
    left_run(120, 120);
    Delay_Ms(200);
}
else if (right_led == 0) {
    // 右侧有障 -> 向左转
    left_run(120, 120);
}
else if (left_led == 0) {
    // 左侧有障 -> 向右转
    right_run(120, 120);
}
else {
    // 两侧都无障碍 -> 前进
    forward(120, 120);
}
```

| 左侧 | 右侧 | 小车动作 | 含义 |
| :---: | :---: | :--- | :--- |
| 1 (无障碍) | 0 (有障) | **左转** | 右侧碰到墙，向左避开 |
| 0 (有障) | 1 (无障碍) | **右转** | 左侧碰到墙，向右避开 |
| 1 (无障碍) | 1 (无障碍) | **前进** | 前方无障碍 |
| 0 (有障) | 0 (有障) | **后退** | 两侧都有障，后退找路 |

### 红外避障 vs 超声波避障

| | 红外避障 | 超声波避障 |
| :--- | :--- | :--- |
| **检测距离** | 2-30cm（近） | 2-400cm（远） |
| **反应速度** | 快（直接读电平） | 慢（需要发射+等待回波） |
| **适用场景** | 贴墙走、精细避障 | 远距离预警 |
| **优点** | 反应快、成本低 | 距离远 |
| **缺点** | 距离近、受颜色影响 | 对软/斜面物体无效 |

> [!TIP]
> **实际小车通常两者配合使用**：超声波远距离预警，红外近距离精细避障。

## 2.2 调试建议

- **传感器不触发**：用手靠近传感器，看模块上 LED 是否亮/灭。如果不亮，调电位器；如果一直亮，可能是模块逻辑反了。
- **转向方向反了**：检查输出逻辑。如果模块是"有障=1"，把代码里的 `== 0` 改成 `== 1`。
- **抖动严重**：在循环中增加 `Delay_Ms(20~50)` 的采样间隔，避免读取太快。
- **检测距离不对**：顺时针调电位器增大距离，逆时针减小。建议从中间位置开始调。

## 本节小结

- 红外避障模块通过发射/接收红外光判断障碍物，输出数字高/低电平
- 本实验使用 OUT 引脚（数字输出），左接 P3^6，右接 P3^7
- 输出逻辑：**本实验测得无障碍=1，有障=0**（不同模块可能不同，以实测为准）
- 避障核心：读传感器 → 判断状态 → 执行动作，循环执行
- 电位器调节检测距离：逆时针增大，顺时针减小
