# 一、前半部分（45 分钟）

## 1.1 红外避障传感器原理

### ① 红外避障的本质

红外避障模块的工作原理其实很直观：模块上的红外发射管持续发光，接收管检测反射回来的光强。当前方有障碍物时，反射光变强，模块输出低电平；无障碍时，反射光弱，输出高电平。

> [!WARNING]
> **不同模块输出逻辑可能相反！** 有些模块是"有障=0"，有些是"有障=1"。接线前务必先验证：用手靠近传感器，看模块上 LED 的亮灭变化，确认输出逻辑。

### ② 模块结构拆解

常见的红外避障模块有 4 个引脚：

| 引脚 | 功能 | 说明 |
| :--- | :--- | :--- |
| VCC | 电源 | 3.3V-5V |
| GND | 地 | 共地 |
| DO | 数字输出 | 比较器输出，0 或 1 |
| AO | 模拟输出 | 反射强度电压（本实验用不到） |

核心是 **LM393 比较器**：红外发射管发光 → 接收管收到反射光 → 产生电压 → 与电位器设定的阈值比较 → 输出高/低。

> [!TIP]
> **电位器的作用**：模块上那个蓝色小旋钮用来调节检测距离。顺时针调灵敏度降低（检测更远），逆时针调灵敏度提高（只能检测很近）。

### ③ 小车上的避障传感器

小车通常安装 **2 个**避障模块，分别检测左右两侧：

| 传感器 | 引脚 | 说明 |
| :--- | :--- | :--- |
| 左避障 | P3^6 | 检测左侧障碍物 |
| 右避障 | P3^7 | 检测右侧障碍物 |

> [!NOTE]
> 引脚名 `left_led2` / `right_led2` 是历史遗留命名，实际功能是红外避障传感器。`led2` 系列用于避障，`led1` 系列用于循迹。

### ④ 实操必看的关键细节

讲完原理，补充几个实操时必须注意的点：

1. **验证输出逻辑**：用手靠近传感器，看模块上 LED 是否亮/灭，确认输出是 0 还是 1。不同批次模块可能逻辑相反！
2. **调电位器**：避障检测距离建议调到 **5-15cm**。太远会提前转向，太近会撞上。
3. **侧面安装角度**：避障模块应朝**侧前方 30-45 度**安装，不要正对侧面，否则反应太晚。
4. **障碍物颜色**：黑色/吸光物体反射弱，检测距离会缩短。白色/反光物体检测距离更远。

## 1.2 避障逻辑

### 核心思路：4 种状态

把左右传感器的读数分开判断，根据状态决定动作：

```c
// 读避障传感器
char left  = left_led2;      // 左：0=有障，1=无障碍
char right = right_led2;     // 右：0=有障，1=无障碍

// 根据状态决定动作
if (right == 0) {
    // 右侧有障 -> 向左转
    left_run(120, 120);
}
else if (left == 0) {
    // 左侧有障 -> 向右转
    right_run(120, 120);
}
else {
    // 两侧都无障碍 -> 前进
    forward(120, 120);
}
```

| 左侧 | 右侧 | 小车动作 | 含义 |
| :---: | :---: | :--- | :--- |
| 1 (无障碍) | 0 (有障) | **左转** | 右侧碰到墙，向左避开 |
| 0 (有障) | 1 (无障碍) | **右转** | 左侧碰到墙，向右避开 |
| 1 (无障碍) | 1 (无障碍) | **前进** | 前方无障碍 |
| 0 (有障) | 0 (有障) | **后退** | 两侧都有障，后退找路 |

### 红外避障 vs 超声波避障

| | 红外避障 | 超声波避障 |
| :--- | :--- | :--- |
| **检测距离** | 2-30cm（近） | 2-400cm（远） |
| **反应速度** | 快（直接读电平） | 慢（需要发射+等待回波） |
| **适用场景** | 贴墙走、精细避障 | 远距离预警 |
| **优点** | 反应快、成本低 | 距离远 |
| **缺点** | 距离近、受颜色影响 | 对软/斜面物体无效 |

> [!TIP]
> **实际小车通常两者配合使用**：超声波远距离预警，红外近距离精细避障。

## 1.3 代码实现

上节课我们学了超声波避障，这节课换用红外模块做避障。相比超声波，红外避障更简单——**直接读电平，不需要发射+等待回波**，所以反应更快。

### 第 1 步：引脚定义

引脚定义在 [config.h](code/config.h) 中：

```c
// config.h
sbit left_led2   = P3^6;   // 左避障传感器（无障碍=1，有障=0）
sbit right_led2  = P3^7;   // 右避障传感器（无障碍=1，有障=0）
```

> [!TIP]
> 避障传感器接在 P3 口，不占用 P1（电机 PWM），互不冲突。

### 第 2 步：读传感器

```c
// 读避障传感器
char left  = left_led2;      // 0=有障，1=无障碍
char right = right_led2;     // 0=有障，1=无障碍
```

> [!NOTE]
> 红外传感器是数字输出，直接 `sbit` 读取即可，不需要定时器或 ADC。

### 第 3 步：判断 + 执行

```c
if (right == 0) {           // 右侧检测到障碍物
    left_run(120, 120);    // 向左转避开
}
else if (left == 0) {       // 左侧检测到障碍物
    right_run(120, 120);   // 向右转避开
}
else {                      // 两侧都无障碍
    forward(120, 120);     // 前进
}
```

## 2.1 完整代码

### 基础版：两侧避障

在 [main.c](code/main.c) 中，完善 `IRAvoid()` 函数：

```c
void IRAvoid(void)
{
    // 1. 读左右避障传感器
    char left  = left_led2;      // 左：0=有障，1=无障碍
    char right = right_led2;     // 右：0=有障，1=无障碍

    // 2. 根据状态决定动作
    if (right == 0) {
        left_run(120, 120);     // 右侧有障 -> 左转
    }
    else if (left == 0) {
        right_run(120, 120);    // 左侧有障 -> 右转
    }
    else {
        forward(120, 120);      // 无障碍 -> 前进
    }
}
```

### 进阶版：双侧同时有障处理

当左右两侧同时检测到障碍物时，增加后退+转向逻辑：

```c
if (right == 0 && left == 0) {
    // 两侧都有障 -> 后退 + 转向
    back_run(120, 120);
    Delay_Ms(300);
    left_run(120, 120);
    Delay_Ms(200);
}
```

### 进阶版：红外+超声波混合避障

结合第 8-9 课的超声波模块，实现远近双重避障：

- **红外**（2-30cm）：侧方障碍物快速反应
- **超声波**（30-300cm）：前方障碍物提前预警

```c
// 伪代码
if (红外检测到侧方障碍) {
    快速转向();            // 贴墙绕行
}
else if (超声波检测到前方障碍) {
    后退 + 转向();         // 脱困
}
else {
    forward();             // 正常前进
}
```

### 进阶版：贴墙走

只用一侧的避障传感器，实现贴墙行走：

```c
// 用右侧避障传感器贴右墙走
if (right_led2 == 0) {
    // 右侧太近 -> 左转远离
    left_run(120, 160);
}
else {
    // 右侧太远 -> 右转靠近
    right_run(160, 120);
}
```

## 2.2 调试建议

- **传感器不触发**：用手靠近传感器，看模块上 LED 是否亮/灭。如果不亮，调电位器；如果一直亮，可能是模块逻辑反了。
- **转向方向反了**：检查输出逻辑。如果模块是"有障=1"，把代码里的 `== 0` 改成 `== 1`。
- **抖动严重**：在循环中增加 `Delay_Ms(20~50)` 的采样间隔，避免读取太快。
- **检测距离不对**：顺时针调电位器增大距离，逆时针减小。建议从中间位置开始调。
