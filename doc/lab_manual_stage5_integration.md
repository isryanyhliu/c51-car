# 实训手册 Stage 5：全系统集成 (智能自主运行)

这是实训的最后一步：将之前学的测距、电机控制、循迹逻辑整合到一个大型工程中。

## 1. 软件工程：模块化开发
为了防止代码混乱，我们建议将核心功能封装成函数。

## 2. 实践：全自动避障 + 串口状态监控
**目标**：实现一个能思考、能行动的小车系统。

```c
#include <reg52.h>
#include "QX_A11.h" // 假设学生已经将引脚定义整理到了 Header 文件中

// 全局状态定义
#define MODE_AVOID 0
#define MODE_TRACK 1
unsigned char Current_Mode = MODE_AVOID;

void System_Init() {
    Timer0_Init(); // 启动 PWM 定时器
    TMOD |= 0x11;  // 同时开启定时器 0 和定时器 1
}

void main() {
    System_Init();
    
    while(1) {
        // 获取距离
        unsigned int dis = Get_Distance();
        
        // 核心决策逻辑
        if(dis < 15) {
            // 紧急避障状态
            Car_Stop();
            Beep_Warning(3); // 响三声
            Car_Back();      // 后退一小段
            Delay_Ms(200);
            Car_Spin_Right(); // 原地掉头
            Delay_Ms(300);
        }
        else {
            // 正常寻路状态：根据循迹传感器走线
            Run_Tracking_Logic(); 
        }
    }
}
```

## 3. 最终调试技巧
1.  **分步调试**：先屏蔽循迹代码，只测避障；避障稳了再加循迹。
2.  **速度匹配**：如果小车避障时反应太慢（撞到了才停），请降低直行时的 `target_speed` (PWM 占空比)。
3.  **功耗注意**：当舵机、电机、超声波同时工作时，电流需求很大。如果程序出现诡异的自动重启，通常是电池没电了。

## 4. 课程结语
从最开始烧录一个“空程序”到实现一个能自主避障、循迹的小车，你已经跨过了嵌入式开发最难的门槛：**理解软件如何与物理定律交织**。
