# 📥 烧录指南：如何将代码送入单片机

在嵌入式开发中，将编译好的 [.hex](file:///c:/Users/isrya/Desktop/20260209_%E6%99%BA%E8%83%BD%E5%B0%8F%E8%BD%A6/code/QX_A11D2%E6%96%B9%E6%A1%88%E5%8F%82%E8%80%83%E7%A8%8B%E5%BA%8F.hex) 文件写入单片机的过程称为“烧录”或“下载”。

## 1. 硬件连接 (Physical Connection)
1. **CH340 模块**：这是一个“翻译官”，负责将电脑的 USB 信号转换为单片机能听懂的串口信号（TTL）。
2. **接线逻辑**：
   - CH340 的 `TXD` 连接单片机的 `RXD` (P3.0)
   - CH340 的 `RXD` 连接单片机的 `TXD` (P3.1)
   - `GND` 必须共地。
   *（注意：如果是成品小车，通常只需要插上一根 USB 线即可，内部已经接好。）*

## 2. 软件配置 (Software Setup)
打开 **STC-ISP** 软件，按照以下顺序操作：
1. **选择型号**：在单片机型号中搜索并选择 `STC89C52RC`。
2. **选择串口**：点击下拉框，选择识别到的 `COM` 口（如 COM3）。如果找不到，请检查 CH340 驱动。
3. **加载文件**：点击“打开程序文件”，选择你在 Keil 中生成的 [.hex](file:///c:/Users/isrya/Desktop/20260209_%E6%99%BA%E8%83%BD%E5%B0%8F%E8%BD%A6/code/QX_A11D2%E6%96%B9%E6%A1%88%E5%8F%82%E8%80%83%E7%A8%8B%E5%BA%8F.hex) 文件。

## 3. 核心机制：冷启动烧录 (The Cold Start)
这是 STC 单片机的特殊机制，也是初学者最容易出错的地方。

1. **点击“下载/编程”按钮**：此时软件会显示“正在检测目标单片机...”。
2. **给小车断电再取电**：立即关闭小车电源开关，然后再打开。
   - **原理**：单片机在程序正常运行时不接受烧录。只有在刚上电的一瞬间，它会先运行一段内置的“引导程序”来检测是否有烧录请求。
3. **完成**：如果看到进度条滚动并提示“操作成功”，则烧录完成。

## 4. 常见问题排查 (Troubleshooting)
- **提示“串口被占用”**：关闭 Keil 的串口调试助手或其他占用该 COM 口的软件。
- **检测不到目标单片机**：
  - 检查接线是否松动。
  - 电池电量是否不足。
  - 确认操作顺序：先点下载，再开电源。

> [!IMPORTANT]
> **什么是 HEX 文件？**
> 这是 C 语言源文件经过编译器（Keil）翻译后的机器码。它是 16 进制文本格式，包含了要写入单片机 Flash 存储器的每一个字节的指令。
