# 🌉 从软件到硬件：嵌入式概念迁移指南

作为一名有编程基础的软件开发者，学习 8051 嵌入式开发时，最核心的转变在于**理解代码是如何直接与物理信号交互的**。

## 1. 执行模型：没有主循环，就没有一切
在 Web 或桌面开发中，你的程序运行在 OS 之上。在 8051 中：
- **Bare Metal**：没有 OS。你的 `main()` 函数永远不能结束。
- **Control Loop**：典型的结构是 `初始化 -> while(1) { 处理任务 }`。
- **Interrupts (中断)**：这是嵌入式版的“事件监听器”。但它是硬件级的抢占。当外部信号或定时器触发时，CPU 强行暂停当前代码，跳转到指定的 `interrupt` 函数。

## 2. 寄存器与 I/O：变量即引脚
在软件中，变量是内存中的值。在嵌入式中，有些特殊的变量（SFR）直接映射到物理引脚：
- `P1 = 0x00;` 实际上是将 P1 端口的所有 8 个引脚电平全部拉低。
- `sbit LED = P1^0;` 定义了一个布尔类型的引脚。当你设置 `LED = 1;` 时，那个物理引脚就会输出 5V（或 3.3V）的电压。

## 3. 内存与指针：物理地址的约束
你可能会在代码中看到 `data`, `xdata`, `pdata`：
- **data/bdata**：高速内部 RAM（非常小，通常只有 128-256 字节）。存放最常用的变量。
- **xdata**：外部扩展 RAM。访问速度较慢。
- **指针**：8051 的指针比通用 C 语言复杂一些，因为编译器需要知道你指向的是哪块物理内存区域。

## 4. 调试思维：信号胜过日志
习惯了 `console.log`？在嵌入式中：
- **断点**：你需要仿真器（如 Keil 的仿真模式或硬件调试器）。
- **信号观测**：有时候最有效的调试手段是用示波器看引脚波形，或者让 `beep` 响一声、`LED` 闪一下来确认代码运行到了哪里。
- **Volatile**：虽然 8051 C 编译器有时会自动处理，但在处理可能会被中断修改的全局变量时，了解原生 C 的 `volatile` 语义（防止编译器过度优化）非常重要。

## 5. 你的代码示例解析
看看你 [main.c](file:///c:/Users/isrya/Desktop/20260209_%E6%99%BA%E8%83%BD%E5%B0%8F%E8%BD%A6/code/main.c) 里的这段：
```c
void timer0_int (void) interrupt 1
{
	 pwm_val_left++;
	 pwm_val_right++;
	 LoadPWM();
}
```
- `interrupt 1`：告诉编译器这是 Timer0 的“事件回调”。
- 这里的代码每隔约 100 微秒就会执行一次。
- `pwm_val_left++`：在极短的时间尺度内计数，从而在 `LoadPWM` 里决定轮子电机的电源开关。

> [!TIP]
> 想像一下你在快速地开关灯。如果你开 1ms 关 1ms，人眼看起来灯是半亮的。这就是你代码里控制小车速度的核心逻辑（PWM）。
