# 实训手册 Stage 1：环境搭建与芯片“清零”

在正式开始编程前，我们需要确保开发环境畅通，并清除芯片内原有的程序，让小车回归到“静止”状态。

## 1. 建立第一个工程：Blank_Program
我们将创建一个最简单的、没有任何逻辑的程序。

### Keil 步骤：
1. 点击 `Project` -> `New µVision Project`。
2. 名字存为 `Blank_Project`。
3. 器件选择 `STC89C52RC` (找不到的同学请安装 STC 数据库)。
4. 创建一个新的 [.c](file:///c:/Users/isrya/Desktop/20260209_%E6%99%BA%E8%83%BD%E5%B0%8F%E8%BD%A6/code/main.c) 文件并添加到工程。

## 2. 示例代码：芯片“清零”程序
这段代码虽然什么都不做，但它是验证烧录路径是否通畅的最佳方式。

```c
/*
 * 实验：芯片清零 / 停止现有一切动作
 * 作用：烧录此程序后，小车电机、声音、灯光应全部静止。
 */
#include <reg52.h>

void main() {
    // 嵌入式程序的“生命循环”
    while(1) {
        // 在死循环里什么都不写
        // CPU 会反复执行空指令，不会触发任何 IO 口的变化
    }
}
```

## 3. 编译与烧录
1. **编译**：点击 Keil 下方的“积木”图标 (Build)。确保输出窗口显示 `0 Error(s), 0 Warning(s)`。
2. **确认 HEX**：在 `Options for Target` -> `Output` 勾选 `Create HEX File`。
3. **烧录**：参考 [烧录指南](file:///C:/Users/isrya/.gemini/antigravity/brain/c30b7e30-3905-4730-8e04-74919a2de910/burning_guide.md) 将生成的 hex 文件下载到小车。

## 4. 实验现象
烧录成功后，小车之前正在跑的所有程序（如避障、音乐）应该立即停止，小车保持彻底静止。

---
## 5. 教师点拨
*   **为什么需要 while(1)？** 
    单片机没有操作系统，如果 `main` 函数结束了，CPU 可能会因为跑飞（PC 指针乱跳）而执行不可控的随机指令。`while(1)` 是嵌入式程序最重要的“地基”。
