# 🚗 智能小车项目技术详解 (开发者版)

既然你已经有了编程基础，我们直接跳过“大脑”和“肌肉”这种简单的比喻，从硬件寄存器映射和控制逻辑的角度来看这个项目。

## 1. 硬件抽象与寄存器映射 ([QX_A11.h](file:///c:/Users/isrya/Desktop/20260209_%E6%99%BA%E8%83%BD%E5%B0%8F%E8%BD%A6/code/QX_A11.h))

在嵌入式开发（特别是 8051）中，你操作的不是抽象的对象，而是硬件引脚。通过 `sbit` 关键字，我们将代码中的变量直接映射到了物理引脚（SFR - 特殊功能寄存器）。

| 代码标识符 | 物理地址/引脚 | 技术说明 |
| :--- | :--- | :--- |
| `EN1`, `EN2` | `P1^4`, `P1^5` | **Enable Pins**: 连接到 L298N/TB6612 电机驱动的使能端。通常用于 PWM 控制速度。 |
| `IN1` ~ `IN4` | `P1^0` ~ `P1^3` | **H-Bridge Control**: 控制电机的正反转。逻辑 `10` 前进，`01` 后退，`00` 停止。 |
| `EchoPin`, `TrigPin` | `P2^0`, `P2^1` | **HC-SR04 接口**: `Trig` 触发脉冲，`Echo` 捕获反射波的高电平持续时间。 |
| `right_led1` 等 | `P3^4` ~ `P3^7` | **红外传感器接口**: 数字信号输入。`0` 表示识别到黑线/障碍物，`1` 表示未识别。 |

## 2. 核心控制逻辑 ([main.c](file:///c:/Users/isrya/Desktop/20260209_%E6%99%BA%E8%83%BD%E5%B0%8F%E8%BD%A6/code/main.c))

### 2.1 运动控制：模拟 PWM
由于这个单片机可能没有硬件 PWM 模块，代码通过 **定时器中断 (Timer0)** 实现了软件 PWM：
- **原理**：在 `timer0_int` 中断服务程序里累加计数值。
- **逻辑**：如果计数值小于设定的 `pwm_left/right`，引脚输出高电平；否则输出低电平。
- **软件层面**：相当于在高频循环中手动控制 `EN1/EN2` 的占空比。

### 2.2 障碍物检测：时间转距离
超声波测距利用了 **定时器1 (Timer1)**：
1. 向 `TrigPin` 发出 10μs+ 的脉冲。
2. 开启 `Timer1` 等待 `EchoPin` 变高。
3. 当 `EchoPin` 回落，读取 `TH1` 和 `TL1` 寄存器的值（计时器步数）。
4. **计算公式**：`Distance = (Time * SpeedOfSound) / 2`。

### 2.3 状态机与任务处理
主循环 `while(1)` 实际上是一个简单的非抢占式任务调度：
- 检查传感器输入（循迹/遥控）。
- 根据状态调用对应的动作函数。
- 处理紧急任务（如 `UltrasonicAvoid` 避障）。

## 3. 嵌入式 vs 软件工程 的差异点

- **无 OS 运行**：没有操作系统进程，整个程序就是一个死循环。
- **实时性**：中断（Interrupt）是处理外部异常（如串口数据、定时器溢出）的核心手段，不能在中断里执行耗时操作（耗时操作会导致主循环卡死）。
- **资源限制**：8051 的内存通常以容量（Byte）计。你看到的 `data`, `idata`, `xdata` 都是在告诉编译器变量存放在哪块物理内存区域。
