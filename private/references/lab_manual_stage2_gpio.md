# 实训手册 Stage 2：GPIO 基础控制 (LED 与 蜂鸣器)

## 1. 理论背景：硬件寄存器映射
在 8051 中，引脚是 8 个一组的（1 个 Byte）。
- `P1` 的地址对应 8 个引脚。
- `sbit` 允许我们将软件中的变量名字绑定到特定的物理引脚位。

## 2. 实践 A：点亮 LED1 (P1.6)
**目标**：理解逻辑电平。

```c
#include <reg52.h>

// 物理映射：将软件标识符 LED1 映射到物理引脚 P1.6
sbit LED1 = P1^6; 

void main() {
    while(1) {
        LED1 = 0; // 低电平：导通，灯亮
    }
}
```

## 3. 实践 B：SOS 报警器 (LED + 蜂鸣器)
**目标**：掌握延时函数与脉冲控制。

```c
#include <reg52.h>

sbit LED1 = P1^6;
sbit BEEP = P2^2; // 蜂鸣器引脚映射

// 简单的毫秒延时函数（受 11.0592MHz 晶振限制）
void Delay_Ms(unsigned int ms) {
    unsigned int i, j;
    for(i = ms; i > 0; i--)
        for(j = 110; j > 0; j--);
}

void main() {
    while(1) {
        LED1 = 0; BEEP = 0; // 同时开启
        Delay_Ms(500);      // 持续 0.5 秒
        
        LED1 = 1; BEEP = 1; // 同时关闭
        Delay_Ms(500);      // 间歇 0.5 秒
    }
}
```

---
## 4. 教师点拨
*   **低电平驱动**：注意很多单片机模块是“低电平有效”（0 = 开启）。让学生观察代码设为 0 时灯是亮还是灭，培养“逻辑 vs 物理”的对应思维。
*   **延时原理**：这里的 `for` 循环其实是在消耗 CPU 的时钟周期。在后续阶段，我们会学习如何用定时器做更精准的计时。
